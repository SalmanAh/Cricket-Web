# Screenshot Guide for PDF Submission

## Important: Sticky Note Requirement
Every screenshot MUST include a visible sticky note (physical or digital) displaying:
- Your Name
- Your Roll Number
- Your Section

You can use a physical sticky note on your screen or add a digital overlay.

---

## Screenshot 1: Aggressive Batting in Action

### Steps:
1. Run the application (`npm start`)
2. Select "Aggressive" batting style (⚡ button)
3. Click "Play Ball"
4. Wait for the power bar to appear
5. Take screenshot showing:
   - Power bar with large RED wicket zone (40%)
   - Moving slider
   - "Aggressive Batting" title
   - Probability labels (0, 0.40, 0.50, etc.)
   - Cricket field in background
   - Scoreboard at top

### What to Highlight:
- The large red "Wicket" segment (40% of bar)
- The purple "6" segment (15% of bar)
- The moving slider indicator

## Screenshot 2: Defensive Batting in Action

### Steps:
1. Select "Defensive" batting style (🛡️ button)
2. Click "Play Ball"
3. Wait for the power bar to appear
4. Take screenshot showing:
   - Power bar with smaller RED wicket zone (20%)
   - Larger gray "0" and green "1" segments
   - "Defensive Batting" title
   - Moving slider
   - Cricket field in background

### What to Highlight:
- The smaller red "Wicket" segment (20% of bar)
- The larger safe zones (0 and 1 runs)
- Comparison with aggressive style

## Screenshot 3: Power Bar Visible During Gameplay

### Steps:
1. Choose any batting style
2. Click "Play Ball"
3. Capture the power bar clearly showing:
   - All color-coded segments
   - Segment labels (OUT, 0, 1, 2, 3, 4, 6)
   - Moving slider with arrow
   - Probability values at bottom (0, 0.40, 0.50, 0.60, 0.70, 0.75, 0.85, 1.00)
   - Instruction text "Click to play your shot!"

### What to Highlight:
- Clear visibility of all segments
- Color coding (Red=Wicket, Gray=0, Green=1, Yellow=2, Orange=3,4, Purple=6)
- Slider position and arrow indicator
- Probability labels

## Screenshot 4: Scoreboard Updates Reflecting Game Progress

### Steps:
1. Play several balls (at least 5-6)
2. Score some runs and maybe lose a wicket
3. Take screenshot showing:
   - Scoreboard with updated values
   - Runs scored (e.g., 15/1)
   - Overs completed (e.g., 0.5 or 1.2)
   - Balls remaining (e.g., 6 or 7)
   - Wickets left (e.g., 1)
   - Cricket field with batsman
   - Commentary box at bottom (if visible)

### Example Good Score:
- Runs: 18/1
- Overs: 1.3
- Balls Left: 9
- Wickets Left: 1

### What to Highlight:
- All four scoreboard sections
- Dynamic values showing game progress
- Commentary text if visible

## Tips for Good Screenshots

1. **Full Screen**: Take full browser window screenshots
2. **Good Timing**: Wait for animations to complete
3. **Clear Visibility**: Ensure all text is readable
4. **Sticky Note**: Make sure your name/roll number is clearly visible
5. **High Resolution**: Use high-quality screenshot tools
6. **No Clutter**: Close unnecessary browser tabs/windows

## Creating PDF Report

### Structure:
1. **Cover Page**
   - Project Title
   - Your Name, Roll Number, Section
   - Course: CS-4032 Web Programming
   - Assignment #02

2. **Screenshots Section**
   - All 5 required screenshots
   - Each with a caption explaining what it shows
   - Sticky note visible in each

3. **Explanation Section**
   - Game Logic and Working Mechanism
   - Mapping of Probabilities to Power Bar
   - Implementation of Animations

4. **Conclusion**
   - Brief summary of project
   - Technologies used
   - GitHub repository link

### Tools for PDF Creation:
- Microsoft Word → Save as PDF
- Google Docs → Download as PDF
- LibreOffice Writer → Export as PDF
- Online tools: Canva, Adobe Spark
